# rename

Ломающий rename без карты call-sites → каскад ошибок + рантайм-разрыв в живом коде (провал,
ради которого скилл и есть). Порядок жёсткий: **найти ссылки → отчёт → атомарная правка.**
Не меняй первым то, на что ссылаются.

## Когда срабатывает — публичная поверхность (есть внешние call-sites)

- **JS/TS** — под `export` (named/default), публичные члены класса
- **Go** — идентификатор с Заглавной (экспорт по правилу компилятора)
- **Python** — module-level без `_`-префикса / перечисленное в `__all__`
- **Плюс** — роут-путь, конфиг-ключ, **переименование/перемещение файла** (путь = символ)

НЕ нужно для локальных переменных / приватного / lowercase-Go / `_`-Python — scope ограничен,
меняй inline (иначе шум на каждый renaming).

## Шаги

1. **Найти все ссылки ДО правки.** Приоритет инструмента:
   - `ast-grep` если доступен — структурный, без ложных срабатываний (отличает символ от слова в комменте).
   - иначе `grep -rn` — fallback; **текстовый, даёт ложные совпадения** → проверяй контекст каждого, не меняй вслепую.
2. **Отчёт blast radius** — «N call-sites в M файлах». Показать команду И вывод, не голый итог.
3. **Атомарная правка** — определение + ВСЕ call-sites одним когерентным заходом, не определение первым.
4. **Проверить** — сборка / тест / линт после (verify, см. `testing.md`).

## Published API

Символ потребляется ВНЕ репозитория (опубликованный пакет, внешние клиенты)? Голый rename = breaking
change (MAJOR semver). Тогда не переименовывай — deprecate + alias на старое имя + minor-релиз.
