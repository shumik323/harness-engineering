# Testing — verify-before-ship

Загружается всегда (нет `paths`). Стек-независимо.

- **Sensor (PostToolUse) запускает тесты сам** при правке в `WATCH_DIR`. Mute the green:
  при успехе молчит, при провале — выводит ошибку. Не дублируй ручным прогоном без нужды.
- **Gate (Stop, Ярус 2) — repo-wide проверка на конце хода, без полных тестов.** `GATE_CMD`
  (type-check + lint + build). Sensor пофайловый: type-ошибки по проекту, линт, сборку,
  поломки конфигов/зависимостей он не видит — это ловит gate. exit 2 → ход не закрыть, пока red.
- **Pre-push (Ярус 3) = gate + полные тесты** (`GATE_TEST_CMD` поверх `GATE_CMD`). Полный
  прогон медленный, на Stop его нет — там тесты закрывает sensor (`TEST_CMD`, пофайлово).
- **Дыра sensor'а:** массовый `lint:fix`/`format` через Bash хук не дёргает PostToolUse —
  после такого прогоняй тесты вручную, иначе поймает только pre-push/CI.
- **Тест = критерий готовности.** Задача не закрыта, пока проверка не зелёная.
- **Осознанно меняешь поведение → обнови ожидания теста/снапшота**, не подгоняй тест
  под баг.
- **Не можешь проверить автоматически → проверь руками** (запуск, скриншот) и скажи об этом
  явно. Не выдавай непроверенное за готовое.
- **Проверяй в состоянии реального использования, не в удобном транзиентном.** Зелёный в
  нерепрезентативном состоянии — ложная уверенность, хуже отсутствия проверки. Сверяй, что среда
  проверки совпадает с тем, как артефакт реально потребляют (released/tagged ref, а не dirty
  worktree; прод-конфиг, а не дев-дефолты; чистая установка, а не твой кэш).
- **Boundary-фича (сеть/API/env/внешний сервис) → перед «готово» объяви уровень verify одной
  строкой:** `verified: real-runtime` ИЛИ честный `verified: tests-only`. Даунгрейд лучше молчания.
  Что значит «реальная среда», см. буллет выше.
