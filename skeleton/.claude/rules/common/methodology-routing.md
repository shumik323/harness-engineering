# Methodology Routing — харнесс по умолчанию

Загружается всегда (нет `paths`). Стек-независимо. CORE-слой.

- **В рабочих проектах дефолт — ТОЛЬКО этот харнесс.** Стороннее (плагин superpowers и др.)
  НЕ запускать без явного запроса пользователя. Рабочие проекты = этот репозиторий и его инстансы.
- **Явный запрос = пользователь назвал инструмент/метод:** «через brainstorming», «subagent'ами»,
  «по TDD», «используй superpowers». Только тогда — можно.
- **Прозрачность.** При запуске стороннего — явно сказать: «внешний плагин X, делает Y».
  Не молча «беру скил».
- **Чекпоинт бьёт continuous.** На необратимом (установка софта, чужой репо, удаление) —
  остановиться и спросить, даже если сторонний скил велит не тормозить.
- **Дубль процесса.** brainstorm/plan ≈ твой SDD-процесс (Specify → Implement → Review). SDD —
  контракт (критерии готовности), сторонний скил — исполнитель. Не дублировать методологию в репо.
- **superpowers — донор идей** для Фаз C/D (self-improve, роли-агенты), не дефолтный процесс.
  Не копировать в репо (дрейф от апстрима).
- **Вне рабочих проектов** (эксперименты, личное, вики) — стороннее ок как дефолт.
