# Workflow — plan → work → review → ship

Загружается всегда (нет `paths`). Универсально для любого стека.

- **Plan first для нетривиального.** Многошаговая задача → план до кода. Однострочный
  diff — без церемоний. План обязан содержать секции из `/plan` (допущения/развилки,
  внешние контракты, teardown); пустая обязательная секция → из планирования не выходим.
- **Spec как контракт.** Для сложного — явные критерии готовности (что должно работать)
  до реализации. Для примитива — типы + тест = спек.
- **Verify before ship.** Любое изменение закрывается проверкой, которую можно запустить
  (тест / сборка / линт). Не можешь проверить — не отправляй. См. `testing.md`.
- **Независимый ревью для нетривиального.** Правка многофайловая / меняет поведение / задевает
  CORE → перед «готово» прогнать **независимый ревью**: built-in `/code-review` или свежий
  ревью-агент. Self-review автора не считается gate — он делит слепые зоны автора (зелёная
  самопроверка ловит не то, что чужой взгляд). Однострочный diff — без церемоний. Нашло
  что-то — чинить (агентом только если правка нетривиальна, иначе руками).
- **Build from failures.** MVP-набор, остальное по реальной боли. Не строить
  инфраструктуру под проблему, которой ещё нет.
- **Наблюдения не теряются.** Заметил нечто стоящее по ходу — `/note` (или прямой
  append в `.claude/PENDING-NOTES.md`). Разбор — в `/end-session`.
- **Завершение = онбординг + как проверить + comprehension-gate.** В конце завершённой
  единицы работы давать: **онбординг** (что сделано, где лежит, как устроено — быстрый вход
  в контекст) и **выжимку «как проверить»** (реально запускаемые команды/шаги, не абстракции).
  Без них задача не закрыта.
- **Comprehension-gate — защита от когнитивного долга.** Единица не закрыта, пока пользователь
  не подтвердил ПОНИМАНИЕ (teach-back: пересказал своими словами), а не только «тесты зелёные».
  Онбординг делит явно: «Понимаешь» / «На доверии (не проверял сам)» — долг виден, не копится молча.
  Фиксировать ПОЧЕМУ, не только что (спека/decisions — против intent debt). Lower-scope по
  умолчанию: давать меньшими порциями, чтобы пользователь успевал рассуждать. Иногда — ремонт
  его рукой без агента (skill preservation).
