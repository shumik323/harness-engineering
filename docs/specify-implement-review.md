# Specify → Implement → Review

Методология работы агента с кодовой базой. Три фазы, жёсткие переходы между ними.

---

## Почему три фазы

Промпт задаёт направление. Спек задаёт критерий готовности.
Без спека агент не знает когда остановиться — он останавливается когда кончается контекст.

```
Промпт → "Сделай компонент авторизации"
Спек   → "Готово когда:
           - пустой email → показать ошибку
           - валидные данные → перейти на /dashboard
           - истёкшая сессия → перейти на /login"
```

Разница: промпт — желание. Спек — контракт. Контракт проверяется автоматически.

---

## Фаза 1 — Specify

**Цель:** сформулировать критерий готовности до первой строки кода.

Форма зависит от сложности:

| Компонент | Форма спека |
|-----------|------------|
| Примитив (кнопка, чип) | Типы + один unit-тест + story. Тест = спек. |
| Средний (dropdown, select) | + явные AC: состояния, edge cases |
| Сложный (stateful, формы) | Формальный GIVEN/WHEN/THEN до кода |

**Выход фазы:** критерий готовности зафиксирован. Агент не переходит к Implement без него.

---

## Фаза 2 — Implement

**Цель:** написать код который удовлетворяет спеку.

Харнесс работает непрерывно:
- Guard блокирует запись в readonly зоны
- Sensor запускает тесты после каждого Edit
- Агент видит красный/зелёный после каждого сохранения

**Принцип mute the green:** sensor молчит при успехе. Сигналит только при провале.
Агент не тратит внимание на шум — только на реальные ошибки.

**Выход фазы:** тесты зелёные, типизация чистая, guard ни разу не сработал.

---

## Фаза 3 — Review

**Цель:** проверить что сделанное соответствует стандартам команды.

```
Проведи ревью изменений, используя правила из @.claude/docs/REVIEW.md
```

Агент загружает REVIEW.md по требованию (JIT) — не при каждом старте сессии.
Прогоняет чеклист. Выдаёт список проблем или подтверждение.

**Выход фазы:** чеклист пройден, PR готов к мержу.

---

## Переходы между фазами

```
Specify  →  критерий зафиксирован           →  Implement
Implement →  тесты зелёные, guard чист       →  Review
Review    →  чеклист пройден                 →  Done
```

Агент не переходит без явного выхода из предыдущей фазы.
Это не автоматика — это дисциплина. Харнесс делает отклонение видимым.

---

## Когда это применимо

Не только миграции. Любая задача где есть критерий готовности — это применимо.

QA: sensor-хуки = quality gate после каждого сохранения, не конца спринта.
Backend: access control работает в любом монорепо.
Greenfield: SDD флоу применим везде.
